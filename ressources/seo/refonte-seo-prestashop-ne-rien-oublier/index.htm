Refondre ou migrer un site sans perdre son trafic SEO représente un défi majeur, parfois même source de stress. Pour certaines entreprises, le référencement naturel est essentiel, et la refonte d’un site web devient alors un enjeu crucial. Nous avons listé ici tous les points de vigilance, illustrés de manière détaillée, si vous envisagez une migration vers le CMS PrestaShop. Certaines particularités sont propres à la solution PrestaShop pour bien gérer votre SEO.

Si vous envisagez de changer de CMS pour passer à PrestaShop, cela constitue déjà un changement significatif. Une approche par étapes peut être avantageuse, en commençant par transférer votre site actuel vers PrestaShop tout en conservant sa structure initiale. Dans un second temps, vous pouvez ajouter les nouvelles fonctionnalités souhaitées pour votre site : un thème différent, une arborescence renouvelée, de nouveaux produits, etc.

Cependant, cette migration en plusieurs étapes n’est pas toujours réalisable. C’est pourquoi nous avons compilé pour vous tous les points essentiels à prendre en compte :

Arborescence SEO dans votre site PrestaShop et URLs

L’arborescence d’un site web est souvent étroitement liée à l’efficacité SEO, en particulier pour les sites de grande envergure. Elle doit être construite de manière cohérente pour le SEO (avec une stratégie en silo ou non, selon vos besoins) et pour offrir une navigation optimale aux utilisateurs. Il est important de noter que l’arborescence d’un site ne se confond pas avec le menu de navigation : ces deux éléments sont bien distincts dans une approche SEO.

Reprise de l’ancienne arborescence e-commerce

Avec le CMS PrestaShop, la transformation de votre site en e-commerce est native, sans besoin de plugins supplémentaires. Vous pouvez directement configurer l’arborescence de votre site pour optimiser la structure de votre catalogue en ligne.

Créer une catégorie de produits dans PrestaShop

Voici comment mettre en place cette arborescence dans PrestaShop pour un site e-commerce. La structure des URLs reflète celle du site et l’organisation de votre catalogue. Par exemple, une URL type pourrait ressembler à ceci : https://monsite.com/categorie-produit-1/produit-1. Commencez par créer les catégories dans PrestaShop, puis attribuez chaque produit à une ou plusieurs de ces catégories pour assurer une navigation fluide et optimisée.

Vous pourrez attribuer à chaque catégorie vos éléments de SEO : balises title, h1 et metadescription. Il est conseillé d’activer rapidement les fonctionnalités SEO de PrestaShop lors de la création de votre site pour une gestion optimale des balises SEO importantes. Nous recommandons de créer les catégories en suivant un ordre hiérarchique : commencez par les catégories principales de l’arborescence, puis descendez progressivement dans les niveaux. Cela vous permettra de définir si une catégorie dépend d’une catégorie parente ou non (à garder en tête pour le Fil d’Ariane) et de structurer ainsi votre arborescence. C’est également à cet endroit que vous pourrez ajouter le texte de description pour chaque catégorie, si nécessaire.

Après avoir créé toutes vos catégories de produits, vous pourrez élaborer vos fiches produits en y intégrant le titre, le contenu texte, les photos, la quantité, etc., puis les rendre visibles en les associant à une catégorie de produits appropriée. Cela permet également d’optimiser le SEO en renseignant correctement les balises essentielles (title, h1 et metadescription). Dans Prestashop, les champs SEO de chaque fiche produit sont directement éditables dans le back-office, vous offrant ainsi un contrôle total sur le référencement de vos pages produits.

Importer des produits en masse grâce au format CSV

Si votre catalogue est conséquent et provient d’une autre boutique ou d’un site sous PrestaShop, vous avez la possibilité d’importer plusieurs produits simultanément grâce à la fonctionnalité d’importation en format CSV.

Gestion des variables : couleur, taille, matière dans les fiches produits

Certains de vos produits peuvent être disponibles en plusieurs couleurs, tailles ou autres variantes. PrestaShop permet de gérer facilement ces variations. En termes de SEO, il est important de s’assurer que chaque variante dispose d’une URL unique ainsi que de balises title, h1, meta description et canonique éditables. Il existe plusieurs méthodes pour optimiser le SEO des variantes de produits. L’idéal est de créer une fiche produit distincte pour chaque variante afin de cibler la longue traîne. Cependant, si cela n’est pas possible, vous pouvez consulter une agence SEO experte comme la nôtre pour obtenir des conseils sur la meilleure approche afin de maintenir l’efficacité SEO avec les variantes de vos produits.

Configurer le tunnel d’achat : prix, expédition, mode de paiement

Bien qu’ils ne soient pas directement liés au SEO, ces éléments sont essentiels pour votre site e-commerce, car ils influencent l’expérience utilisateur. Une configuration optimale du prix, des options d’expédition et des modes de paiement contribue à rendre l’expérience d’achat fluide et agréable pour vos clients.

Si vous avez un grand nombre d’articles de blog à reporter dans votre nouveau back-office sous WordPress, il existe des modules qui permettent d’importer en volume des articles de blog grâce à la création d’un fichier XML, WXR ou CSV de ces articles. Si vous importez des articles depuis un autre site sous Prestashop avec le même module de blog, alors l’import sera plutôt simple. En revanche, si vous souhaitez importer des articles de blog depuis un autre CMS que Prestashop, alors il vous faudra utiliser des plugins qui permettent d’importer un fichier lisible.

Souvent, ces modules utilisent le flux RSS, ce dernier représente souvent un langage ccompréhensible entre les CMS. Ainsi, les données associées aux articles de blog (titre, contenu, date de publication, auteur, image, format, champ personnalisé, etc.) seront conservées lors de l’import, ce qui représente une bonne continuité de la prise en compte par les robots d’indexation.

Les URLs de vos images, vidéos et autres contenus média

Lorsque vous importez vos médias dans la bibliothèque, le CMS génère automatiquement une nouvelle URL dédiée pour chacun de vos fichiers. Cependant, cette URL peut différer de celle utilisée sur votre site en production. Cette distinction doit être prise en compte lors de la création de votre fichier de redirections. Nous vous conseillons de rediriger les anciennes URLs des images vers les nouvelles, car les images peuvent contribuer au trafic de votre site, notamment via Google Images. Vous pouvez vérifier l’impact de ce trafic dans la Search Console de votre site. Si vos images génèrent un volume significatif de clics, il est essentiel d’inclure leurs redirections dans votre fichier de redirections (voir le paragraphe dédié).

Refonte avec changement de nom de domaine

Si vous changez à la fois de domaine et de CMS, il est recommandé de procéder à la refonte en deux étapes.

Étape 1 : Reproduisez votre site tel quel sur le nouveau CMS tout en configurant toutes les redirections nécessaires avec le nouveau nom de domaine. N’oubliez pas, en cas de changement de nom de domaine, de transférer l’ancienne Search Console vers la nouvelle. Ce transfert peut être signalé dans les paramètres de la Search Console, ce qui facilite la prise en compte des modifications par Google.

Étape 2 : Une fois les performances de l’ancien site au maximum sur le nouveau domaine, procédez à la refonte complète de votre site (arborescence, design, contenu, etc.).

Votre nouveau site est-il responsive ?

Pour rester en accord avec les tendances actuelles en matière de design, de performance et d’expérience utilisateur, il est essentiel que votre site soit adapté à toutes les tailles d’écran (mobile, ordinateur de bureau, tablette, etc.). Un site capable de s’ajuster automatiquement à ces formats est qualifié de “responsive design”. Bien que la majorité des thèmes téléchargeables ou achetables pour votre site soient conçus pour être responsives, il est important de le vérifier.

Depuis juillet 2024, Google a officialisé la fin de sa transition vers le modèle d’indexation Mobile-first (initiée en 2016 !). Cela signifie que désormais, tous les sites seront analysés en priorité par des robots d’indexation optimisés pour les smartphones (Googlebot smartphone). Par conséquent, avoir un site mobile-friendly est devenu indispensable pour optimiser votre SEO.

L’utilisation du JS dans le thème de site

De nombreux thèmes disponibles pour Prestashop sont optimisés pour le SEO. Cependant, il est crucial de vérifier si le thème que vous choisissez n’est pas entièrement basé sur le JavaScript. Bien que les robots d’indexation progressent dans la compréhension de ce langage, il peut encore poser des problèmes pour l’analyse et l’indexation de vos pages ou de leur contenu. Nous vous recommandons de consulter attentivement la documentation associée aux thèmes que vous envisagez d’utiliser, car cette information y est souvent mentionnée.

Soyez également vigilant avec les plugins de construction de pages (Page Builder). Certaines fonctionnalités de ces outils utilisent le JavaScript de manière peu optimale pour le SEO. Pour éviter toute erreur ou perte de performance, nous vous conseillons de faire appel à un expert SEO, qui pourra vous guider dans la création de vos pages et lors de la refonte de votre site.

Recettage SEO : avant la mise en ligne et après la mise en ligne

La phase de recettage d’un nouveau site est essentielle pour le SEO, tout comme le fichier de redirections. Elle garantit que tous les aspects techniques et sémantiques sont correctement intégrés dans le nouveau site. Cette étape permet d’identifier et de corriger les éventuels blocages, afin d’assurer la continuité de l’indexation et la bonne prise en compte des contenus après la mise en ligne. Voici les principaux points à vérifier lors du recettage :

  • Arborescence
  • Structure des URLs
  • Nommage SEO des pages (balises title, h1, metadescription)
  • Implantation des contenus SEO (structure des balises <hn>, texte, images, vidéos, etc.)
  • Accessibilité des pages (maillage interne, menu de navigation, etc.) : éviter les pages orphelines ou inaccessibles aux robots d’indexation et aux utilisateurs.
  • Analyse technique avec un outil de crawl

Après la mise en ligne, une nouvelle recette est indispensable pour vérifier que tous les éléments ont été correctement appliqués lors de la bascule.

Dans les jours suivants, il est crucial de surveiller le comportement des robots d’indexation via la Search Console. Cela permet de vérifier que la transition est bien comprise (prise en compte des redirections, suivi des positions sur les mots-clés, etc.).

Les redirections des URLs : un essentiel du succès de votre refonte

La refonte de votre e-commerce peut être comparée à faire visiter une nouvelle maison au robot d’indexation, notamment si vous avez changé de nom de domaine et/ou modifié les URLs et leur structure. Le robot conserve en mémoire (dans son index) toutes les anciennes URLs de votre site, et sans un fichier de redirections, il sera incapable de retrouver la nouvelle version de ces pages.

Pour lui faciliter la tâche – et accélérer l’indexation des nouvelles URLs, ce qui améliore votre efficacité SEO –, il est essentiel de créer un fichier de redirections. Ce fichier sera intégré par votre webmaster ou développeur dans :

  • L’ancien site, en cas de changement de nom de domaine.
  • Le nouveau site, si le nom de domaine reste inchangé.

Sans ce fichier de redirections, le robot d’indexation pourrait mettre beaucoup de temps à réindexer votre site, ce qui pourrait prendre plusieurs semaines, voire plusieurs mois. Cette période d’attente peut représenter un risque pour certaines entreprises.

Un fichier de redirection est simplement une instruction qui indique que l’URL X doit rediriger vers l’URL Y.

Ce fichier de redirections est vital pour garantir l’efficacité de votre refonte.

Nous vous conseillons de réaliser ce fichier de redirections après le recettage de la pre-production, lorsqu’elle cette dernière ne subira plus aucun changement.

Vous trouverez sur ce lien tous nos conseils pour utiliser correctement les redirections 301 en SEO.

Le cas des redirections si vous ne changez pas de domaine :

Si certaines URLs de votre site sont supprimées ou modifiées lors de la refonte, il est essentiel de planifier leurs redirections. La redirection 301, qui est permanente, est la plus adaptée et parfaitement comprise par Google. Si l’une de ces URLs générait du trafic ou jouait un rôle clé dans vos performances SEO, veillez à ce que le contenu de la page d’origine soit correctement intégré dans la nouvelle page correspondante.

Rappelez-vous que c’est la qualité du contenu texte qui détermine l’efficacité d’une page. Les redirections doivent être configurées dans le fichier .htaccess de votre site, généralement situé à la racine de celui-ci.

Le cas des redirections si vous changez de nom de domaine :

Si vous prévoyez de migrer d’abord vers un nouveau nom de domaine sans modifier le reste du site, vous devrez configurer des redirections de toutes les URLs de l’ancien site vers leurs équivalents sur le nouveau domaine.

En revanche, si vous optez pour une migration complète en une seule étape (changement de nom de domaine + refonte du site), il est crucial de prêter une attention particulière à la création du fichier de redirections. Ce fichier devra être placé sur le site de l’ancien domaine.

Pour cette raison, il est important de ne pas supprimer immédiatement l’ancien site. Vous devez laisser à Google le temps de traiter et d’intégrer correctement toutes les redirections, ce qui peut prendre entre 3 et 6 mois en moyenne, si tout est bien configuré.

L’intérêt du fichier .htaccess :

Le fichier .htaccess est un outil interprété par les robots d’indexation, conçu pour ajuster le comportement d’un serveur web. Il permet de configurer diverses règles, telles que la sécurité, la gestion du cache ou encore les redirections. C’est également dans ce fichier que seront définies et spécifiées les redirections que vous souhaitez mettre en place.

Module de gestion des redirections sur PrestaShop

Si vous ne souhaitez pas toucher à ce fichier vous-même parce que vous n’êtes pas sûr de vous, vous pouvez utiliser un plugin qui va gérer l’encodage des redirections à votre place. Il vous faut cependant tester le plugin afin d’implanter toutes vos redirections. Certains plugins peuvent être incompatibles avec votre site, votre serveur, votre thème ou simplement pas mis à jour depuis longtemps.

Vous trouverez les modules de gestion des redirections pour prestaShop ici.

Surveillance de la bonne prise en compte de vos redirection dans la Search Console

Une fois que le nouveau site est en ligne et que le fichier de redirections est correctement configuré, nous vous recommandons de tester vous-même les redirections. Pour ce faire, entrez une ancienne URL dans un navigateur et observez le résultat : si la redirection fonctionne correctement, l’URL devrait changer (à vous de vérifier que c’est bien l’URL de destination que vous avez définie). Si cela ne se produit pas, cela signifie que la redirection n’a pas été prise en compte (vous pourriez rencontrer une erreur 404 ou être redirigé vers l’ancienne page du site).

Il est également important de suivre les données remontées dans la Search Console pour vérifier que les redirections sont bien traitées (allez dans le volet Pages > Pages avec redirections). Si des problèmes d’indexation persistent ou si les nouvelles URLs ne sont pas correctement prises en compte, une agence SEO spécialisée pourra identifier la source du problème et vous apporter l’aide nécessaire.

Tester les différentes fonctionnalités de votre site après la mise en ligne

Bien qu’ils ne soient pas directement liés au SEO, ces éléments peuvent bloquer l’expérience utilisateur s’ils ne sont pas fonctionnels et efficaces :

  • Reconnexion de tous les services tiers (plugins de mesure d’audience, réseaux sociaux, cache, clés de licence, etc.)
  • Formulaire de contact
  • Tunnel d’achat
  • Redirections

Si vous avez changé de domaine, il est également essentiel de mettre à jour votre fiche Google My Business (GMB), les liens vers votre site sur vos réseaux sociaux, ainsi que votre communication globale.

Pourquoi cet article n’aborde pas… ?

La refonte d’un site web englobe plusieurs aspects qui ne sont pas nécessairement liés au SEO. C’est pourquoi nous avons uniquement abordé les éléments en lien avec le SEO. Cependant, il est probable que vous ayez d’autres tâches à accomplir ou que vous deviez faire appel à des experts pour certains domaines spécifiques :

  • Wireframes/Design : Cette tâche incombe généralement aux web designers, qui définissent la structure des pages. Le SEO doit vérifier que le wireframe comprend les espaces nécessaires pour le contenu SEO (texte, images, titres, sous-titres, etc.).
  • Cahier des charges technique : Un consultant SEO peut vous aider à rédiger le cahier des charges techniques nécessaires aux développeurs, afin que le site respecte les critères SEO. Une agence SEO PrestaShop, comme la nôtre, peut vous accompagner dans la préparation de ce cahier des charges. Cependant, il doit être élaboré avant le démarrage du projet, idéalement lors de la phase pré-production.
  • Tracking analytics : Si votre site dispose déjà de tags de suivi, assurez-vous qu’ils sont correctement intégrés sur le nouveau site, afin que vos données soient continuellement collectées dans vos outils de suivi et d’analyse.

Faire une copie de votre site actuel peut vous sauver

Nous vous recommandons de conserver une copie de votre ancien site, afin de pouvoir revenir en arrière si la migration ne se déroule pas comme prévu. Un développeur expérimenté pourra vous garantir que cela sera possible si nécessaire.
